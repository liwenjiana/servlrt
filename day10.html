1.过滤器的优点

a.不修改源代码的情况下，为程序增加新的功能。　
b.可以将多个web组件相同的逻辑集中写在一个类里面
（即过滤器里面），方便代码的维护。

2.jsp基础

(1)jsp是什么?

sun公司制订的一种服务器端的动态页面技术规范。
本质就是一个servlet。

(2)如何写一个jsp文件?

step1.写一个以.jsp为后缀的文件。
step2.添加
 1)html(css,js): 直接写
 2)java代码:
    a. java代码片断
        <%  java代码  %>
    b. jsp表达式
        <%= java表达式 %>
    c. jsp声明     (a1.jsp)
        <%! 声明一个变量或者方法  %>
 3)指令
    a.什么是指令?
     告诉容器，在将jsp文件转换成servlet类的时候，做一些额外的处理（比如导包）。
    b.语法
        <%@ 指令名　　属性＝值 %>
    c.page指令
        import属性：导包
        pageEncoding属性：告诉容器，在读取jsp
            文件的内容时，使用哪个字符集去解码。
        contentType属性：设置response.setContentType方法的参数值。
        session属性: (a2.jsp) 缺省值是true,
                如果值为false,session隐含对象
                就不能用了。
        errorPage属性 (a3.jsp):指定一个异常处理
            页面。当前页面发生异常，容器会调用该异常处理页面来处理。
        isErrorPage属性 (a4.jsp):缺省值是 false,如果值为true,则可以使用
            exception隐含对象。

    d.include指令
        file属性：指定要插入的文件的名称。
        容器在将jsp转换成servlet类时，会将
        指定的文件的内容插入到该指令所在的位置。

    c.taglib指令
4)隐含对象
    a.什么是隐含对象?
    可以直接使用的对象，比如out
    b.为什么可以直接使用这些隐含对象?
    容器会添加获得这些隐含对象的代码。
    c.有哪一些隐含对象?
        out,request,response
        session
        application
        exception
        pageContext: (a6.jsp,a7.jsp) 
            容器会为每一个jsp实例创建唯一的一个
            符合PageContext接口要求的对象，
            该对象会一直存在（除非jsp实例被销毁）。
            作用１：绑订数据
                注：绑订的数据只有对应的jsp
                能够访问（相当于jsp实例的钱包）。
            作用２：提供了查找其它所有隐含对象
                的方法。

        config (a5.jsp):就是ServletConfig对象 
        page (jsp实例本身) 
            注:jsp对应的servlet对象
5)注释  (a8.jsp)
    a. <!-- 注释内容 --> 如果注释的内容是java
            代码，仍然会执行。
    b. <%-- 注释内容 --%>

(3)jsp是如何执行的?

阶段一：　容器要将jsp文件转换成一个servlet类。
    html ---->service方法里，使用out.write输出
    <% %> --->service方法里，照搬。
    <%= %> -->service方法里，使用out.print输出
    <%!  %>--->添加新的属性或者方法。
阶段二：　容器调用该servlet。

3. jsp标签和el表达式

(1)jsp标签是什么?

jsp标签语法类似于html标签，用来替换jsp中的java
代码。
注：
    因为直接在jsp文件中写java代码，不利于页面的
    维护（比如将jsp文件交给美工去修改就不方便） 。
    使用jsp标签代替jsp文件中的java代码，有两个
    优点：
        a.jsp文件变得简洁。
        b.jsp标签可以复用。

(2)el表达式是什么?

一套简单的运算规则，用于给jsp标签的属性赋值，
也可以直接输出（即也可以脱离jsp标签直接使用）。

(3)el表达式的使用

1)访问bean的属性
注：如果一类满足如下几个要求，就可以称之为一个
    javabean。
    a.public class
    b.有无参构造器 (public)。
    c.有一些属性及对应的get/set方法。
方式一 
    ${user.username}
        容器依次从pageContext,request,session,
        servletContext中查找绑订名为"user"的对象(即执行getAttribute方法)，找到之后，
        调用该对象的getUsername方法并输出。
        注:
            a.会将null转换成""输出。
            b.如果依据绑订名找不到对应的
            对象，会输出""。
            c.可以使用pageScope,requestScope,
            sessionScope,applicationScope
            来指定查找范围。
方式二 
    ${user['username']} 
        注:
            a.[]里面可以使用绑订名。
            b.[]里面可以使用从０开始的下标，
            用来访问数组中指定下标的元素。
2) 进行一些简单的运算，运算结果可以给jsp
    标签的属性赋值，也可以直接输出。
   a. 算术运算　+,-,*,/,%
        注: 
            + 只能求和。
   b.关系运算  >,<,>=,<=,==,!=
   c.逻辑运算　&&,||,!
   d,empty运算　empty

3)读取请求参数值     
    ${param.uname}
    等价于
        String request.getParameter("uname");
    ${paramValues.city}
    等价于
        String[] request.getParameterValue("city");
